# novellm

Claude Code를 집필 파트너로 활용하는 **AI 협업 소설 창작 시스템**입니다.

세계관·인물·플롯을 구조화된 폴더로 관리하고, Claude가 git 히스토리만으로 소설 전체 맥락을 파악해 일관성 있는 집필·퇴고를 보조합니다.

---

## 핵심 아이디어

| 문제 | 해결 방식 |
|------|----------|
| AI가 이전 대화 내용을 잊는다 | `canon/` · `state/` · `story/` 폴더로 컨텍스트를 파일로 영속화 |
| 설정이 화마다 어긋난다 | `canon/`(고정 법칙)과 `state/`(가변 현황)를 분리해 충돌 방지 |
| 복선을 심고 잊어버린다 | 복선 ID 체계 + `복선-목록.md` / `복선-회수표.md`로 추적 |
| AI가 세계 상태를 파악 못 한다 | `[AI-CONTEXT]` 블록이 포함된 구조화 커밋 메시지로 `git log` 가독성 확보 |

---

## 폴더 구조

```
소설-프로젝트/
├── CLAUDE.md              ← Claude Code 자동 로드 (AI 진입점)
├── .claude/               ← AI 역할·규칙·현재 작업 컨텍스트
│
├── canon/                 ← 고정 설정 (변경 금지)
│   ├── 세계관/            ← 능력체계, 사회문화, 역사, 지리
│   ├── 지도/              ← ASCII 세계지도 + 좌표계
│   ├── 아이템/
│   ├── 몬스터/
│   └── 주제/              ← 테마, 톤앤매너, 상징
│
├── state/                 ← 가변 상태 (화 진행에 따라 업데이트)
│   ├── 인물/              ← 주연·조연 현재 능력치·관계도
│   ├── 세력/
│   ├── 지도/              ← 탐험 현황·거리표
│   ├── 몬스터/            ← 등장 개체·보스 생존 현황
│   ├── 아이템/
│   └── 복선/              ← 복선 목록 + 회수 추적
│
├── story/                 ← 스토리 구성
│   ├── 플롯/              ← 전체흐름, 감정곡선
│   └── 에피소드/          ← 화별 기획서
│
└── output/원고/           ← 최종 완성 원고
```

## 커밋 형식

Claude가 `git log`만으로 소설 상태를 추적할 수 있도록 모든 커밋에 `[AI-CONTEXT]` 블록을 포함합니다.

**세계관/설정 커밋**
```
world(능력체계): 각성 등급 S~F 6단계 분류 추가

[AI-CONTEXT]
추가요소: 각성 등급 체계, 등급별 능력 상한선
영향범위: 능력-체계.md, 인물/주인공.md
연관복선: 주인공이 F등급 판정을 받는 1화 복선 뒷받침
[/AI-CONTEXT]
```

**에피소드 커밋**
```
ep(1화): 주인공, F등급 각성 판정을 받다

[AI-CONTEXT]
등장인물: 주인공 이준혁, 심사관 오성민
핵심사건: 각성 검사에서 최하등급 F 판정. 주변의 조롱 속에 홀로 귀가.
심은복선: 검사 중 심사관이 순간 당황하는 표정 (실제 등급 은폐 암시)
[/AI-CONTEXT]
```

## 시작하는 법

1. 이 레포를 클론합니다.
2. `소설-프로젝트/` 폴더를 Claude Code로 엽니다 — `CLAUDE.md`가 자동 로드됩니다.
3. `canon/세계관/개요.md`부터 세계 기초 공사를 시작합니다.
4. 자세한 워크플로우는 [`소설-프로젝트/README.md`](소설-프로젝트/README.md)를 참고하세요.
