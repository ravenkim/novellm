# novellm

Claude Code를 집필 파트너로 활용하는 **AI 협업 집필 시스템**입니다.  
아래 **구조별 특징**에 맞는 폴더 하나를 골라 그 안에서만 작업하면 됩니다.

---

## 공통으로 갖는 것

| 요소 | 역할 |
|------|------|
| `CLAUDE.md` | Claude Code가 자동으로 읽는 AI 진입점 |
| `.claude/` | 역할·규칙(`system.md`, `rules.md`)·현재 작업(`context/`)·프롬프트 양식 |
| `canon/` | **고정 축** — 바꾸면 작품 전체의 약속·정의가 흔들리는 내용 |
| `state/` | **가변 축** — 집필이 진행될수록 갱신되는 현황·추적 정보 |
| `story/` | **계획 축** — 목차·화(장) 설계·요약 |
| `output/원고/` | 완성 원고 |
| Git 커밋의 `[AI-CONTEXT]` | `git log`만으로 맥락을 이어 갈 수 있게 하는 짧은 구조화 메타데이터 |

**공통으로 쓰는 해결 전략:** AI가 대화를 잃어버려도 파일에 맥락이 남고, `canon`과 `state`를 나눠 설정 충돌을 줄이며, 커밋에 맥락을 심어 히스토리로 복원한다.

## 구조별 특징

|  | **소설** [`소설-프로젝트/`](소설-프로젝트/) | **수필·에세이·자기개발** [`수필-자기개발-프로젝트/`](수필-자기개발-프로젝트/) | **학술 논문** [`논문-프로젝트/`](논문-프로젝트/) |
|--|---------------------------------------------|-----------------------------------------------------------------------------|------------------------------------------|
| **한 줄** | 허구 서사 — 세계와 인물이 시간에 따라 변한다 | 비픽션 — 한 권의 **논지**와 **독자 약속**이 축이다 | 학술 글 — **연구 질문·기여**와 **방법·인용·형식**이 축이다 |
| **`canon/`에 두는 것** | 세계의 법칙·지리·역사, 용어, 톤·테마 등 **작품 안에서 이미 정해진 진실** | 핵심 논지, 저자 약속, 독자 프로필, 용어 정의, 톤, **사실·인용 규칙** | RQ·기여·novelty, 타깃 학회·저널·분량, 용어·약어, **방법 한계**, **인용 스타일** |
| **`state/`에 두는 것** | 인물·세력·장소의 **현재** 상태, 복선 심기·회수, 정보·사건 인덱스 | 집필 진도, **사례**가 어디에 쓰였는지, 반복 메시지, **팩트체크 대기** | 집필 진도, **인용·수치 검증 큐**, 동료 검토·**재심 응답**, 실험·**재현 메모** |
| **`story/`에 두는 것** | 플롯(전체흐름·감정곡선), **에피소드(화)별** 기획, (선택) 화 요약 | **목차**, 독자의 인지·저항 흐름, **챕터** 기획, 장별 한 줄 요약 | 논문 골격(IMRaD 등), **심사자·독자 기대**, **섹션** 기획, 섹션별 한 줄 요약 |
| **일관성을 잡는 도구** | 복선 ID, 인물 시트, 세계관 제약, (장르에 따라) 지도·아이템·몬스터 등 | 사례 은행, 메시지 일관성, 인용·검증 큐 | 기여–결과 정렬, 방법–본문 대응, 검토 의견 추적표 |
| **첫으로 채우기 좋은 파일** | `canon/세계관/개요.md` → `canon/주제/테마.md` 등 (해당 작품에 맞는 `canon`부터) | `canon/핵심-논지.md`, `canon/저자-약속.md`, `canon/독자-프로필.md` | `canon/연구-질문-기여.md` → `canon/저널-양식.md`, `canon/인용-스타일.md` |
| **커밋 규칙** | `world`, `char`, `ep`, `plot` 등 — [`소설-프로젝트/.claude/rules.md`](소설-프로젝트/.claude/rules.md) | `thesis`, `ch`, `voice`, `fact` 등 — [`수필-자기개발-프로젝트/.claude/rules.md`](수필-자기개발-프로젝트/.claude/rules.md) | `rq`, `sec`, `venue`, `cite`, `review` 등 — [`논문-프로젝트/.claude/rules.md`](논문-프로젝트/.claude/rules.md) |

세부 워크플로우·파일 설명은 각 폴더의 `README.md`를 보면 된다.

## 폴더 트리 (요약)

### 소설 — `소설-프로젝트/`

```
소설-프로젝트/
├── CLAUDE.md
├── .claude/
├── canon/
│   ├── 세계관/          ← 개요, 능력·사회·역사·지리 등 (작품에 맞게 사용)
│   ├── 지도/            ← (필요 시) 세계·좌표
│   ├── 아이템/ · 몬스터/ …
│   └── 주제/              ← 테마, 톤앤매너, 상징
├── state/
│   ├── 인물/ · 세력/ · 복선/ · …   ← 진행에 따라 바뀌는 것
│   └── …
├── story/
│   ├── 플롯/
│   └── 에피소드/          ← 화별 기획
└── output/원고/
```

레포에 들어 있는 예시는 장르 도구(지도·몬스터 등)가 포함되어 있다. 현실주의 소설만 쓸 경우 해당 폴더는 비우거나 쓰지 않으면 된다.

```

├── .claude/

│   ├── 핵심-논지.md · 저자-약속.md · 독자-프로필.md
│   ├── 톤앤매너.md · 용어집.md · 인용-근거-규칙.md
├── state/
```

## 커밋 형식 (맛보기)

프로젝트마다 `[AI-CONTEXT]` 필드 이름이 조금 다르다.

**소설** — 설정 예시:

```
world(능력체계): …

[AI-CONTEXT]
추가요소: …
영향범위: …
연관복선: …
[/AI-CONTEXT]
```

**소설** — 한 화 예시:

```
ep(1화): …

[AI-CONTEXT]
등장인물: …
핵심사건: …
심은복선: …
[/AI-CONTEXT]
```

**수필·자기개발** — 한 장 예시는 [`수필-자기개발-프로젝트/.claude/rules.md`](수필-자기개발-프로젝트/.claude/rules.md)의 `ch(<N장>)` 블록을 따른다.

**학술 논문** — 섹션 예시는 [`논문-프로젝트/.claude/rules.md`](논문-프로젝트/.claude/rules.md)의 `sec(<섹션명 또는 번호>)` 블록을 따른다.

## 시작하는 법

1. 이 레포를 클론한다.
2. **소설**이면 `소설-프로젝트/`, **수필·자기개발**이면 `수필-자기개발-프로젝트/`, **학술 논문**이면 `논문-프로젝트/`를 Claude Code로 연다 — 해당 폴더의 `CLAUDE.md`가 로드된다.
3. 위 표의 **첫으로 채우기 좋은 파일**부터 채운다.
4. 장면·장·섹션 단위 작업 흐름은 각 프로젝트 `README.md`를 따른다.
