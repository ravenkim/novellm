# novellm

Claude Code를 집필 파트너로 활용하는 **AI 협업 집필 시스템**입니다.

프로젝트 종류별로 폴더가 나뉩니다.

| 폴더 | 용도 |
|------|------|
| [`소설-프로젝트/`](소설-프로젝트/) | **소설(허구)** — 세계관·인물·플롯·복선 중심 |
| [`수필-자기개발-프로젝트/`](수필-자기개발-프로젝트/) | **수필·에세이·자기개발서** — 논지·독자 약속·사례·목차 중심 |

각각 `canon/`(고정) · `state/`(가변) · 집필 계획 폴더로 컨텍스트를 파일로 남기고, `[AI-CONTEXT]`가 포함된 커밋으로 git만으로도 맥락을 추적합니다.

---

## 핵심 아이디어

| 문제 | 해결 방식 |
|------|----------|
| AI가 이전 대화 내용을 잊는다 | `canon/` · `state/` · `story/` 폴더로 컨텍스트를 파일로 영속화 |
| 글이 장·화마다 어긋난다 | `canon/`(고정)과 `state/`(가변)를 분리해 충돌 방지 |
| 소설: 복선 추적 / 비소설: 논지·사실·사례 일관성 | 소설은 복선 ID. 수필·자기개발은 `핵심-논지`·`사례-은행`·`인용-검증-대기` |
| AI가 전체 맥락을 못 잡는다 | `[AI-CONTEXT]`가 들어간 구조화 커밋으로 `git log` 가독성 확보 |

## 폴더 구조 (소설: `소설-프로젝트/`)

```
소설-프로젝트/
├── CLAUDE.md              ← Claude Code 자동 로드 (AI 진입점)
├── .claude/               ← AI 역할·규칙·현재 작업 컨텍스트
│
├── canon/                 ← 고정 설정 (변경 금지)
│   ├── 세계관/            ← 능력체계, 사회문화, 역사, 지리
│   ├── 지도/              ← ASCII 세계지도 + 좌표계
│   ├── 아이템/
│   ├── 몬스터/
│   └── 주제/              ← 테마, 톤앤매너, 상징
│
├── state/                 ← 가변 상태 (화 진행에 따라 업데이트)
│   ├── 인물/              ← 주연·조연 현재 능력치·관계도
│   ├── 세력/
│   ├── 지도/              ← 탐험 현황·거리표
│   ├── 몬스터/            ← 등장 개체·보스 생존 현황
│   ├── 아이템/
│   └── 복선/              ← 복선 목록 + 회수 추적
│
├── story/                 ← 스토리 구성
│   ├── 플롯/              ← 전체흐름, 감정곡선
│   └── 에피소드/          ← 화별 기획서
│
└── output/원고/           ← 최종 완성 원고
```

수필·자기개발서용 레이아웃은 [`수필-자기개발-프로젝트/README.md`](수필-자기개발-프로젝트/README.md)를 참고하세요.

## 커밋 형식

프로젝트마다 `[AI-CONTEXT]` 필드가 조금 다릅니다. **소설** 예시:

**세계관/설정**
```
world(능력체계): 각성 등급 S~F 6단계 분류 추가

[AI-CONTEXT]
추가요소: …
영향범위: …
연관복선: …
[/AI-CONTEXT]
```

**에피소드**
```
ep(1화): …

[AI-CONTEXT]
등장인물: …
핵심사건: …
심은복선: …
[/AI-CONTEXT]
```

**수필·자기개발** 예시는 [`수필-자기개발-프로젝트/.claude/rules.md`](수필-자기개발-프로젝트/.claude/rules.md)를 참고하세요.

## 시작하는 법

1. 이 레포를 클론합니다.
2. **소설**이면 `소설-프로젝트/`, **수필·자기개발서**면 `수필-자기개발-프로젝트/`를 Claude Code로 엽니다 — 각 폴더의 `CLAUDE.md`가 자동 로드됩니다.
3. 소설: `canon/세계관/개요.md`부터. 수필·자기개발: `canon/핵심-논지.md` · `canon/저자-약속.md`부터 채웁니다.
4. 세부 워크플로우는 각 폴더의 `README.md`를 참고하세요.
