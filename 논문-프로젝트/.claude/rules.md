# 작성 규칙

## 논리·기여 일관성
- 연구 질문·기여·novelty는 `canon/연구-질문-기여.md` 기준
- 다루지 않을 주제·방법 한계는 `canon/방법론-경계.md`에 따름
- 핵심 용어는 `canon/용어-정의.md`로 통일

## 사실과 인용
- 검증이 필요한 수치·선행연구·직접 인용은 `state/인용-검증-대기.md`에 올리거나 확인 후 반영
- 각주·참고문헌 형식은 `canon/인용-스타일.md` 준수

## 섹션 작성 후 처리
- 완성된 섹션은 `story/요약/섹션-목차.md`에 한 줄 요약 추가
- 방법·결과 변경 시 `state/실험-재현-메모.md` 동기화
- 리비전·재심 대응은 `state/검토-응답-대기.md`에 반영
- `context/current.md` 업데이트

## 폴더별 수정 권한
| 폴더 | 수정 가능 여부 | 설명 |
|------|--------------|------|
| `canon/` | **제한** | RQ·기여·용어·방법 경계·인용 규칙. 변경 시 논문 전체 영향 |
| `state/` | **자유** | 진행, 검증 큐, 검토 응답, 실험 메모 |
| `story/` | **자유** | 골격·섹션 계획 |
| `output/` | **자유** | 원고 |

## 커밋 규칙

### 타입 분류

#### 연구·구조 계열
| 타입 | 용도 |
|------|------|
| `rq` | 연구 질문·가설·기여 문구 변경 |
| `method` | 방법론 경계·재현 메모·분석 범위 |
| `venue` | 타깃 저널·분량·형식 |
| `cite` | 인용·참고문헌 규칙 |
| `term` | 용어 정의 |
| `outline` | 논문 골격·목차 |
| `fix` | 오류·모순 수정 |

#### 원고 계열
| 타입 | 용도 |
|------|------|
| `sec` | 특정 섹션 초안·완료 |
| `abstract` | 초록 |
| `draft` | 전체 초안 마일스톤 |
| `revise` | 퇴고·리비전 |
| `review` | 동료/심사 의견 반영 |
