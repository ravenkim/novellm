# 학술 논문 프로젝트 — 사용 가이드

> **용도:** 학회·저널 논문, 학위 논문의 **본문 편집·구조화**. 연구 질문·기여·방법 경계·인용을 한축으로 묶습니다.  
> 소설은 [`소설-프로젝트/`](../소설-프로젝트/README.md), 에세이·자기개발은 [`수필-자기개발-프로젝트/`](../수필-자기개발-프로젝트/README.md)를 사용하세요.

## 폴더 구조 원칙

| 폴더 | 변경 여부 | 담는 것 |
|------|----------|---------|
| `canon/` | **제한적** | RQ·기여, 타깃 저널·분량, 용어, 방법 경계, 인용 스타일 |
| `state/` | **자유** | 집필 진도, 인용 검증 큐, 동료 검토·재심 응답, 실험·재현 메모 |
| `story/` | **자유** | 논문 골격(IMRaD 등), 심사자 관점 메모, 섹션 기획 |
| `output/` | **자유** | 완성 원고 |
| `.claude/` | — | AI 역할, 규칙, 현재 작업 컨텍스트 |

## 작성하면 좋은 순서

| 순서 | 하는 일 | 주로 쓰는 위치 |
|:----:|---------|----------------|
| 1 | RQ·기여·novelty를 고정한다 | `canon/연구-질문-기여.md` |
| 2 | 타깃 학회·저널·분량·언어를 적는다 | `canon/저널-양식.md` · `canon/인용-스타일.md` |
| 3 | 용어·약어·방법 한계를 정리한다 | `canon/용어-정의.md` · `canon/방법론-경계.md` |
| 4 | 전체 골격(IMRaD 등)과 그림·표 목록을 잡는다 | `story/논문-골격.md` |
| 5 | 심사 관점에서 빈틈을 적는다 | `story/독자-심사자-여정.md` |
| 6 | 섹션별 집필 루프 | `story/섹션/_템플릿.md` 복사 → `섹션/<이름>.md` · `output/원고/` · `.claude/context/current.md` |
| 7 | 섹션 완료 후 | `story/요약/섹션-목차.md` · `state/집필-진행.md` · 미확인 인용은 `state/인용-검증-대기.md` · 실험 수치는 `state/실험-재현-메모.md` |

## canon vs state 판단

> **"바뀌면 논문의 주장·방법 정의·학술적 정체성이 흔들리는가?"**
> - YES → `canon/`
> - NO → `state/`

| `canon/` 예시 | `state/` 예시 |
|--------------|--------------|
| RQ·기여 문구 | 지금 Methods 초안 몇 페이지인지 |
| 인용 스타일 | 아직 bibtex 안 단 인용 |
| 용어 정의 | 리뷰어 R2 의견 대응표 |

## Claude Code

`논문-프로젝트/`에서 Claude Code를 열면 `CLAUDE.md`가 로드되며 `system.md` · `rules.md` · `context/*.md`가 이어집니다.

## Git 커밋 요약

RQ·골격·저널 형식은 `rq` · `outline` · `venue` 등, 섹션 완성은 `sec` — 자세한 형식은 `.claude/rules.md`를 따릅니다.
