# 작성 규칙

## 설정 일관성
- 인물의 이름, 외형, 말투는 인물 파일 기준으로 고정
- 세계관 고유 용어는 용어집.md 기준으로 통일
- 능력/아이템 효과는 능력-체계.md 범위 내에서만 사용

## 복선 관리
- 새 복선을 심으면 → `state/복선/복선-목록.md`에 추가
- 복선을 회수하면 → `state/복선/복선-회수표.md`에 표시

## 에피소드 작성 후 처리
- 완성된 화는 `story/요약/`에 한 줄 요약 추가
- 인물 능력치·관계 변화 → `state/인물/` 해당 파일에 반영
- 새 세계관 정보(법칙·역사) → `canon/` — 수정 전 작가 확인 필수
- `context/current.md` 업데이트

## 폴더별 수정 권한
| 폴더 | 수정 가능 여부 | 설명 |
|------|--------------|------|
| `canon/` | **금지** | 세계의 법칙·정의. 오류 발견 시 `fix:` 커밋으로만 수정 |
| `state/` | **자유** | 스토리 진행에 따라 적극 업데이트 |
| `story/` | **자유** | 플롯·에피소드 계획 |
| `output/` | **자유** | 완성 원고 추가 |

## 커밋 규칙

### 타입 분류

커밋은 두 가지 계열로 구분한다.

#### 세계관/설정 계열
| 타입 | 용도 |
|------|------|
| `world` | 세계관 설정 추가/수정 (역사, 문화, 사회, 능력체계 등) |
| `char` | 인물 추가/수정 |
| `map` | 지리·장소·지도 추가/수정 |
| `monster` | 몬스터 추가/수정 |
| `lore` | 용어집·상징·모티프 추가/수정 |
| `plot` | 플롯·전체흐름·복선 구성 변경 |
| `fix` | 설정 오류·모순 수정 |

#### 에피소드 계열
| 타입 | 용도 |
|------|------|
| `ep` | 새 화(話) 완성 |
| `draft` | 화 초안 저장 |
| `revise` | 기존 화 퇴고/수정 |

### 커밋 메시지 형식

#### 세계관/설정 커밋
```
<타입>(<대상>): <한 줄 요약>

[AI-CONTEXT]
추가요소: <새로 추가된 핵심 정보>
영향범위: <변경으로 영향받는 기존 설정 파일 또는 항목>
연관복선: <이 설정이 심거나 뒷받침하는 복선 (없으면 생략)>
[/AI-CONTEXT]
```

예시:
```
world(능력체계): 각성 등급 S~F 6단계 분류 추가

[AI-CONTEXT]
추가요소: 각성 등급 체계, 등급별 능력 상한선
영향범위: 능력-체계.md, 인물/주인공.md
연관복선: 주인공이 F등급 판정을 받는 1화 복선 뒷받침
[/AI-CONTEXT]
```

---

#### 에피소드 커밋
```
ep(<N화>): <제목 또는 핵심 사건 한 줄>

[AI-CONTEXT]
등장인물: <이 화에 등장한 인물 목록>
핵심사건: <줄거리 한두 문장>
심은복선: <새로 심은 복선 (없으면 생략)>
회수복선: <이 화에서 회수된 복선 (없으면 생략)>
신규설정: <이 화에서 드러난 새 세계관 정보 (없으면 생략)>
[/AI-CONTEXT]
```

예시:
```
ep(1화): 주인공, F등급 각성 판정을 받다

[AI-CONTEXT]
등장인물: 주인공 이준혁, 심사관 오성민
핵심사건: 각성 검사에서 최하등급 F 판정. 주변의 조롱 속에 홀로 귀가.
심은복선: 검사 중 심사관이 순간 당황하는 표정 (실제 등급 은폐 암시)
회수복선: 없음
신규설정: 각성 센터 묘사, 검사 절차
[/AI-CONTEXT]
```

---

### 규칙 요약
- `[AI-CONTEXT]` 블록은 AI가 `git log`만으로 소설 상태를 파악하기 위한 구조화 데이터다. 반드시 포함한다.
- 세계관과 에피소드는 **별도 커밋**으로 분리한다. 하나의 커밋에 섞지 않는다.
- 에피소드 완성 전 중간 저장은 `draft(N화):`를 사용한다.
- 설정 오류 수정 시 `fix(<파일명>):`으로 영향 범위를 명시한다.
