# 소설 프로젝트 — 사용 가이드

> **용도:** **소설(허구 서사)** 집필에 맞춘 템플릿입니다. 세계관·인물·복선·지도 등 서사 일관성을 전제로 합니다.  
> 수필·에세이·자기개발서는 상위 디렉터리의 [`수필-자기개발-프로젝트/`](../수필-자기개발-프로젝트/README.md)를 사용하세요.

## 폴더 구조 원칙

폴더는 **변경 가능 여부**로 구분한다.

| 폴더 | 변경 여부 | 담는 것 |
|------|----------|---------|
| `canon/` | **금지** | 세계의 법칙·규칙·역사·정의 (아이템 규칙, 능력체계, 지형 등) |
| `state/` | **자유** | 스토리 진행에 따라 변하는 현재 상태 (캐릭터 능력치, 관계도, 세력 상황 등) |
| `story/` | **자유** | 플롯 계획, 에피소드 기획, 완성화 요약 |
| `output/` | **자유** | 최종 완성 원고 |
| `.claude/` | — | AI 컨텍스트 (역할·규칙·현재 작업) |

## 작성하면 좋은 순서

아래 번호가 작을수록 먼저 한다. **4~6**은 완결까지 반복하는 **화 집필 루프**다.

| 순서 | 하는 일 | 주로 쓰는 위치 |
|:----:|---------|----------------|
| 1 | 세계의 전제와 작품의 목소리를 고정한다 | `canon/세계관/개요.md` → `canon/주제/테마.md` → `canon/주제/톤앤매너.md` → 작품에 필요하면 `능력-체계`, `사회-문화`, `역사-연표`, `용어집`, `지도/`, `아이템/`, `몬스터/` 등 (**안 쓰는 장르 요소는 건너뛴다**) |
| 2 | 이야기가 시작될 때의 인물·세력 상태를 둔다 | `state/세력/세력-개요.md` → `state/인물/주연/`·`조연/` (`_템플릿.md` 복사) |
| 3 | 큰 줄기와 독자 감정의 흐름을 잡는다 | `story/플롯/전체흐름.md` → (선택) `story/플롯/감정곡선.md` |
| 4 | 집필할 화부터 에피소드 기획서를 만든다 | `story/에피소드/_템플릿.md` 복사 → `NN화.md` · `.claude/context/current.md` 갱신 |
| 5 | 원고를 쓴다 | `output/원고/` · `.claude/prompt/template.md` 양식으로 요청해도 됨 |
| 6 | 그 화를 파일에 반영한다 | `story/요약/` 한 줄 요약 → `state/복선/` · `state/인물/` 등 → `.claude/context/current.md` · `summary.md` → `git commit` (메시지 형식은 이 문서 하단 **Git 커밋 규칙** 참고) |
| 7 | 다음 화로 돌아가 **4**부터 반복한다 | — |

자세한 단계 설명·예시는 이 문서 아래 **워크플로우** 절을 보면 된다.

## canon vs state 판단 기준

새 파일/내용을 어디에 넣을지 모를 때 이 질문으로 판단한다.

> **"이 소설이 끝날 때까지 바뀌지 않는 세계의 규칙인가?"**
> - YES → `canon/`
> - NO → `state/`

| `canon/` 예시 | `state/` 예시 |
|--------------|--------------|
| 능력 등급 체계 (S~F) | 주인공의 현재 등급 |
| 아이템 종류와 효과 규칙 | 주인공이 소지한 아이템 목록 |
| 지형 (산맥·강·바다 위치) | 주인공이 탐험한 지역 현황 |
| 세계의 역사·연표 | 세력 간 현재 관계·갈등 |
| 몬스터 분류 규칙 | 특정 보스의 처치/생존 상태 |
| 톤앤매너·테마 | 인물 관계도 (화마다 변함) |

## Claude Code 자동 인식 원리

`소설-프로젝트/` 폴더에서 Claude Code를 열면 `CLAUDE.md`가 자동 로드됩니다.

```
Claude Code 열기
    → CLAUDE.md 자동 로드
        → system.md  (AI 역할 지침 + 폴더 원칙)
        → rules.md   (복선·커밋 규칙 + 폴더별 수정 권한)
        → context/current.md  (지금 어떤 화 작업 중인지)
```

## 워크플로우

### 1단계 — 세계 기초 공사 (처음 한 번, canon/ 작성)

| 순서 | 파일 | 무엇을 쓰나 |
|------|------|------------|
| 1 | `canon/세계관/개요.md` | 세계의 한 줄 설명, 핵심 전제 |
| 2 | `canon/주제/테마.md` | 이 소설이 말하려는 것 |
| 3 | `canon/주제/톤앤매너.md` | 시점, 문장 스타일 |
| 4 | `canon/세계관/능력-체계.md` | 능력/마법 규칙 |
| 5 | `state/세력/세력-개요.md` | 주요 세력 구도 (초기값) |
| 6 | `state/인물/주연/[이름].md` | `_템플릿.md` 복사해서 작성 |
| 7 | `story/플롯/전체흐름.md` | 기승전결 큰 그림 |

### 2단계 — 에피소드 기획

```
story/에피소드/_템플릿.md 복사
    → story/에피소드/03화.md 로 저장
    → 씬 구성, 등장 인물, 심을 복선 작성
    → .claude/context/current.md 업데이트
```

### 3단계 — 집필 요청

`.claude/prompt/template.md`의 양식을 채워 Claude에게 전달:

```
[3화] 집필 요청
- 이 화의 목표: 주인공이 처음으로 능력을 자각하는 장면
- 시작 상황: 시장 골목, 추격 중
- 끝내야 할 지점: 능력 발현 후 혼자 남겨짐
- 강조할 감정/톤: 혼란 → 경이로움
```

### 4단계 — 집필 후 정리 (state/ 업데이트)

화가 완성되면 순서대로:

1. `story/요약/03화-요약.md` — 한 줄 요약 작성
2. `state/복선/복선-목록.md` — 심은 복선 추가
3. `state/복선/복선-회수표.md` — 회수한 복선 체크
4. `state/인물/` — 능력치·관계 변화 반영
5. `.claude/context/current.md` — 현재 작업 화 업데이트
6. `.claude/context/summary.md` — 누적 요약 테이블에 추가
7. git commit

## 인물 파일 만드는 법

```bash
# 주연 (풀 시트 — 목표·동기·두려움·성장호 포함)
cp state/인물/주연/_템플릿.md state/인물/주연/이름.md

# 조연 (기능 중심 — 서사 역할과 관계만)
cp state/인물/조연/_템플릿.md state/인물/조연/이름.md
```

## 복선 관리

| 파일 | 역할 |
|------|------|
| `state/복선/복선-목록.md` | 심은 복선 전체 목록 (ID: F001, F002...) |
| `state/복선/복선-회수표.md` | 회수 여부 추적 |

심을 때: `복선-목록.md`에 ID와 함께 추가
회수할 때: `복선-회수표.md`에 회수 화와 방식 기록

## 지도 시스템

### 고정 vs 가변

| 고정 (`canon/지도/`) | 가변 (`state/지도/`) |
|--------------------|-------------------|
| 세계지도 (대륙·바다 위치) | 탐험 현황 |
| 좌표계 (1칸 = 몇 km) | 도시·거점 현재 상태 |
| 지형 (산맥·강·특수지형) | 거리-이동표 (경로 개척에 따라 추가) |

### 좌표 표기

```
열(A~Z) + 행(1~99) = 셀 좌표
예) C4 = C열 4행 / C4.3 = C열 4행 세로 3/10 지점
```

### 새 장소 등록 체크리스트
- [ ] `canon/지도/세계지도.md` 해당 셀에 기호 추가
- [ ] `state/지도/대륙/` 또는 `지역/` 파일에 목록 추가
- [ ] `state/지도/도시-거점/` 또는 `canon/지도/지형/` 에 상세 파일 작성
- [ ] `state/지도/거리-이동표.md` 에 다른 거점과의 거리 추가
- [ ] `canon/세계관/지리-장소.md` 에 서사적 의미 연결

## 몬스터 시스템

### 고정 vs 가변

| 고정 (`canon/몬스터/`) | 가변 (`state/몬스터/`) |
|----------------------|----------------------|
| 분류체계 (등급 규칙) | 등장 개체·보스·종족 현황 |

### 개체/종족/보스 구분

| 폴더 | 언제 쓰나 |
|------|---------|
| `state/몬스터/개체/` | 이름 없는 일반 몬스터, 특정 개체 |
| `state/몬스터/종족/` | 같은 특성을 공유하는 군집 단위 |
| `state/몬스터/보스/` | 고유명이 있거나 페이즈가 있는 핵심 전투 상대 |

### 새 몬스터 등록 체크리스트
- [ ] `state/몬스터/목록.md` 인덱스에 추가
- [ ] 해당 `_템플릿.md` 복사 후 이름으로 저장
- [ ] `canon/몬스터/분류체계.md` 등급 기준 확인
- [ ] 서식지 좌표를 `canon/지도/세계지도.md`와 연결
- [ ] 드랍 아이템은 `canon/아이템/아이템-목록.md`에 추가
- [ ] 보스라면 복선 연결 항목 작성

## Git 커밋 규칙

### 세계관/설정 커밋
```
<타입>(<대상>): <한 줄 요약>

[AI-CONTEXT]
추가요소: ...
영향범위: ...
연관복선: ...
[/AI-CONTEXT]
```

| 타입 | 용도 |
|------|------|
| `world` | canon/세계관 추가/수정 |
| `char` | state/인물 추가/수정 |
| `map` | 지도 추가/수정 |
| `monster` | 몬스터 추가/수정 |
| `lore` | 용어집·상징 추가/수정 |
| `plot` | 플롯·전체흐름 변경 |
| `fix` | 설정 오류·모순 수정 |

### 에피소드 커밋
```
ep(<N화>): <제목 또는 핵심 사건 한 줄>

[AI-CONTEXT]
등장인물: ...
핵심사건: ...
심은복선: ...
회수복선: ...
신규설정: ...
[/AI-CONTEXT]
```

| 타입 | 용도 |
|------|------|
| `ep` | 새 화 완성 |
| `draft` | 화 초안 저장 |
| `revise` | 기존 화 퇴고/수정 |

## 자주 쓰는 흐름 요약

