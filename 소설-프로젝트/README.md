## Claude Code 자동 인식 원리

`소설-프로젝트/` 폴더에서 Claude Code를 열면 `CLAUDE.md`가 자동으로 로드됩니다.  
`CLAUDE.md`는 내부적으로 `@.claude/system.md`, `@.claude/rules.md`, `@.claude/context/current.md`를 임포트하므로, **별도 설명 없이도 AI가 이 소설의 설정과 현재 작업을 인식**합니다.

```
Claude Code 열기
    → CLAUDE.md 자동 로드
        → system.md  (AI 역할 지침)
        → rules.md   (복선·커밋 규칙)
        → context/current.md  (지금 어떤 화 작업 중인지)
```

## 워크플로우

### 1단계 — 세계 기초 공사 (처음 한 번)

순서대로 채워두면 나중에 AI가 혼자 참조합니다.

| 순서 | 파일 | 무엇을 쓰나 |
|------|------|------------|
| 1 | `세계관/개요.md` | 세계의 한 줄 설명, 핵심 전제 |
| 2 | `주제/테마.md` | 이 소설이 말하려는 것 |
| 3 | `주제/톤앤매너.md` | 시점, 문장 스타일 |
| 4 | `세계관/능력-체계.md` | 마법/능력 규칙 |
| 5 | `세계관/세력/세력-개요.md` | 주요 세력 구도 |
| 6 | `인물/주연/[이름].md` | `_템플릿.md` 복사해서 작성 |
| 7 | `플롯/전체흐름.md` | 기승전결 큰 그림 |

### 2단계 — 에피소드 기획

```
플롯/에피소드/_템플릿.md 복사
    → 플롯/에피소드/03화.md 로 저장
    → 씬 구성, 등장 인물, 심을 복선 작성
    → .claude/context/current.md 업데이트
```

### 3단계 — 집필 요청

`.claude/prompt/template.md`의 양식을 채워 Claude에게 전달:

```
[3화] 집필 요청
- 이 화의 목표: 주인공이 처음으로 능력을 자각하는 장면
- 시작 상황: 시장 골목, 추격 중
- 끝내야 할 지점: 능력 발현 후 혼자 남겨짐
- 강조할 감정/톤: 혼란 → 경이로움
```

### 4단계 — 집필 후 정리

화가 완성되면 순서대로:

1. `요약/03화-요약.md` — 한 줄 요약 작성
2. `소재/복선-목록.md` — 심은 복선 추가
3. `플롯/복선-회수표.md` — 회수한 복선 체크
4. `.claude/context/current.md` — 현재 작업 화 업데이트
5. `.claude/context/summary.md` — 누적 요약 테이블에 추가
6. git commit

## 인물 파일 만드는 법

### 주연 (풀 시트)
```bash
# _템플릿.md를 복사해 이름으로 저장
cp 인물/주연/_템플릿.md 인물/주연/이름.md
```
주연 시트에는 **목표·동기·두려움·성장 호**까지 작성합니다.  
빌런도 주연 폴더에 넣되, 역할 항목에 `빌런`으로 표시하세요.

### 조연 (기능 중심 시트)
```bash
cp 인물/조연/_템플릿.md 인물/조연/이름.md
```
조연은 **서사 기능**(정보제공, 코믹릴리프 등)과 주인공과의 관계만 간결하게 씁니다.

## 복선 관리

복선은 두 파일로 이중 관리합니다.

| 파일 | 역할 |
|------|------|
| `소재/복선-목록.md` | 심은 복선 전체 목록 (ID 부여) |
| `플롯/복선-회수표.md` | 회수 여부 추적 |

심을 때: `복선-목록.md`에 ID(F001, F002...) 와 함께 추가  
회수할 때: `복선-회수표.md`에 회수 화와 방식 기록

## 지도 시스템 사용법

### 지도 구축 순서

| 순서 | 파일 | 무엇을 정하나 |
|------|------|-------------|
| 1 | `지도/좌표계.md` | 1칸 = 몇 km인지, 이동 수단별 속도 정의 |
| 2 | `지도/세계지도.md` | ASCII 그리드에 대륙·바다 윤곽 그리기 |
| 3 | `지도/대륙/_템플릿.md` 복사 | 대륙별 상세 지도·특징 작성 |
| 4 | `지도/지역/_템플릿.md` 복사 | 대륙 안 지역 단위 세분화 |
| 5 | `지도/도시-거점/_템플릿.md` 복사 | 주요 도시·던전 등록 |
| 6 | `지도/지형/` | 산맥·강·특수지형 기록 |
| 7 | `지도/거리-이동표.md` | 거점 간 이동 시간 계산 |

### 좌표 표기 방법

```
열(A~Z) + 행(1~99) = 셀 좌표
예) C4 = C열 4행

정밀 위치가 필요하면 소수점 사용
예) C4.3 = C열 4행 안에서 세로 3/10 지점
```

### ASCII 지도에서 지형 기호 채우는 법

```
     A    B    C    D
   +----+----+----+----+
 3 | ~~ | ~~ | ^^ | :: |   ← 바다·바다·산맥·평원
   +----+----+----+----+
 4 | ~~ | :: |@수도| :: |   ← 바다·평원·수도·평원
   +----+----+----+----+
```

기호: `~~` 바다 / `^^` 산맥 / `::` 평원 / `**` 숲 / `##` 사막 / `@` 도시 / `!` 특수장소

### 새 장소 등록 체크리스트
- [ ] `세계지도.md` 해당 셀에 기호 추가
- [ ] 대륙·지역 파일에 목록 추가
- [ ] `도시-거점/` 또는 `지형/` 에 상세 파일 작성
- [ ] `거리-이동표.md` 에 다른 거점과의 거리 추가
- [ ] `세계관/지리-장소.md` 에 서사적 의미 연결

## 몬스터 시스템 사용법

### 개체 / 종족 / 보스 구분 기준
| 파일 위치 | 언제 쓰나 |
|----------|---------|
| `개체/` | 이름 없는 일반 몬스터, 특정 개체 |
| `종족/` | 같은 특성을 공유하는 군집 단위 (예: 석화고블린 종족) |
| `보스/` | 고유명이 있거나 페이즈가 있는 핵심 전투 상대 |

### 새 몬스터 등록 체크리스트
- [ ] `몬스터/목록.md` 인덱스에 추가
- [ ] 해당 `_템플릿.md` 복사 후 이름으로 저장
- [ ] `분류체계.md`의 등급 기준 확인
- [ ] 서식지 좌표를 `지도/세계지도.md`와 연결
- [ ] 드랍 소재는 `소재/아이템/아이템-목록.md`에 추가
- [ ] 보스라면 복선 연결 항목 작성

### 보스 템플릿 핵심 항목
```
페이즈 구성  →  전투 흐름  →  처치 후 밝혀지는 것
```
보스는 단순 장애물이 아닌 이야기의 일부로 설계합니다.  
처치 이후 주인공에게 어떤 변화가 생기는지까지 미리 정해두세요.

## Git 커밋 규칙

| 접두사 | 사용 상황 | 예시 |
|--------|----------|------|
| `feat:` | 새 화 완성 | `feat: 3화 완성` |
| `plan:` | 플롯/구성 변경 | `plan: 2막 전환점 수정` |
| `update:` | 기존 설정 보완 | `update: 주인공 백스토리 추가` |
| `fix:` | 설정 오류/모순 수정 | `fix: 능력 체계 상충 수정` |
| `refactor:` | 구조 정리 (내용 변경 없음) | `refactor: 인물 파일 분리` |
