# 작성 규칙

## 논지·톤 일관성
- 책 전체의 한 문장 논지는 `canon/핵심-논지.md` 기준
- 독자에게 하지 않을 말·다루지 않을 주제는 `canon/저자-약속.md`에 따름
- 핵심 용어는 `canon/용어집.md`로 통일

## 사실과 의견
- 검증이 필요한 통계·연구·인용은 `state/인용-검증-대기.md`에 올리거나 확인 후 반영
- 개인 경험님은 사실이 아닌 맥락으로 명시 (시점·한계)

## 챕터 작성 후 처리
- 완성된 장은 `story/요약/목차.md`에 한 줄 요약 추가
- 사용한 사례는 `state/사례-은행/` 해당 항목에 사용처 표시
- 반복 메시지 변경 시 `state/메시지-일관성.md` 업데이트
- `context/current.md` 업데이트

## 폴더별 수정 권한
| 폴더 | 수정 가능 여부 | 설명 |
|------|--------------|------|
| `canon/` | **제한** | 논지·약속·용어·근거 규칙. 변경 시 전 책 영향 — 의도적 수정만 |
| `state/` | **자유** | 진행 상황, 사례, 검증 큐 |
| `story/` | **자유** | 목차·챕터 계획 |
| `output/` | **자유** | 원고 |

## 커밋 규칙

### 타입 분류

#### 논지·입장 계열
| 타입 | 용도 |
|------|------|
| `thesis` | 핵심 논지·논점 구조 변경 |
| `voice` | 톤앤매너·용어집 |
| `promise` | 저자 약속·독자 프로필 |
| `fact` | 사실·출처·근거 규칙 |
| `outline` | 목차·독자 여정 |
| `fix` | 오류·모순 수정 |

#### 원고 계열
| 타입 | 용도 |
|------|------|
| `ch` | 챕터 완성 |
| `draft` | 초안 저장 |
| `revise` | 퇴고·수정 |

### 커밋 메시지 형식

#### 논지·구조 커밋
```
<타입>(<대상>): <한 줄 요약>

[AI-CONTEXT]
추가요소: <새로 확정한 내용>
영향범위: <영향받는 장·사례·메시지>
주의사항: <사실 검증·윤리적 경계 (없으면 생략)>
[/AI-CONTEXT]
```

#### 챕터 커밋
```
ch(<N장>): <제목 또는 핵심 메시지 한 줄>

[AI-CONTEXT]
핵심메시지: ...
사용사례: ...
다음장연결: ...
검증필요: ...
[/AI-CONTEXT]
```
