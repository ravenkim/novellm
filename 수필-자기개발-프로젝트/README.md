# 수필·자기개발 프로젝트 — 사용 가이드

> **용도:** 수필, 에세이, 자기개발·비픽션 계열. **한 권의 논지**와 **독자 약속**을 중심에 둡니다.  
> 소설(허구)은 [`소설-프로젝트/`](../소설-프로젝트/README.md)를 사용하세요.

## 폴더 구조 원칙

| 폴더 | 변경 여부 | 담는 것 |
|------|----------|---------|
| `canon/` | **제한적** | 한 문장 논지, 저자 약속, 독자 프로필, 용어, 톤, 사실·인용 규칙 |
| `state/` | **자유** | 집필 진행, 사례 사용처, 반복 메시지, 팩트체크 대기 |
| `story/` | **자유** | 목차, 독자 여정, 챕터 기획, 장별 요약 |
| `output/` | **자유** | 완성 원고 |
| `.claude/` | — | AI 역할, 규칙, 현재 작업 컨텍스트 |

## 작성하면 좋은 순서

아래 번호가 작을수록 먼저 한다. **4~6**은 완결까지 반복하는 **장 집필 루프**다.

| 순서 | 하는 일 | 주로 쓰는 위치 |
|:----:|---------|----------------|
| 1 | 한 권의 논지와 저자·독자 관계를 고정한다 | `canon/핵심-논지.md` → `canon/저자-약속.md` · `canon/독자-프로필.md` → `canon/톤앤매너.md` → `canon/용어집.md` · `canon/인용-근거-규칙.md` (사실·인용이 많을수록 **1**에서 빨리 잡는다) |
| 2 | 목차와 독자의 인지·저항 흐름을 잡는다 | `story/목차-전체.md` · `story/독자-여정.md` |
| 3 | 쓸 이야기·사례를 카드로 쌓는다 | `state/사례-은행/` (`_템플릿.md` 복사) · `state/집필-진행.md` 초기 표기 |
| 4 | 집필할 장부터 챕터 기획서를 만든다 | `story/챕터/_템플릿.md` 복사 → `챕터/NN-제목.md` · `.claude/context/current.md` 갱신 |
| 5 | 원고를 쓴다 | `output/원고/` · `.claude/prompt/template.md` 양식으로 요청해도 됨 |
| 6 | 그 장을 추적 정보에 반영한다 | `story/요약/목차.md` → `state/집필-진행.md` · `state/메시지-일관성.md` → 불확실 팩트는 `state/인용-검증-대기.md` → `.claude/context/summary.md` → `git commit` (타입·`[AI-CONTEXT]`는 **Git 커밋 요약** 및 `.claude/rules.md`) |
| 7 | 다음 장으로 돌아가 **4**부터 반복한다 | — |

아래 **워크플로우** 절과 같은 줄기이며, 표는 순서만 한눈에 보기 쉽게 정리한 것이다.

## canon vs state 판단

> **"이 책이 출간본 기준으로 바뀌면 안 되는 약속·정의인가?"**
> - YES → `canon/`
> - NO → `state/`

| `canon/` 예시 | `state/` 예시 |
|--------------|--------------|
| 핵심 논지·지지 논점 | 지금 몇 장 초안 완료인지 |
| 의료·법률 조언을 하지 않겠다는 약속 | 아직 팩트체크 안 한 문장 목록 |
| 용어 정의 | 사례 A를 3장에서 이미 썼는지 |
| 인용·각주 스타일 | 검증 대기 큐 |

## 워크플로우

### 1단계 — 뼈대 (처음 한 번)

| 순서 | 파일 |
|------|------|
| 1 | `canon/핵심-논지.md` |
| 2 | `canon/저자-약속.md` · `canon/독자-프로필.md` |
| 3 | `canon/톤앤매너.md` |
| 4 | `story/목차-전체.md` · `story/독자-여정.md` |

### 2단계 — 챕터 기획

```
story/챕터/_템플릿.md 복사 → story/챕터/01-제목.md
state/사례-은행/ 에 필요한 사례 카드 추가
.claude/context/current.md 업데이트
```

### 3단계 — 집필·정리

1. `output/원고/`에 초안
2. `story/요약/목차.md`에 한 줄 요약
3. `state/집필-진행.md` · `메시지-일관성.md` 반영
4. 불확실한 팩트 → `state/인용-검증-대기.md`
5. `.claude/context/summary.md` 테이블 갱신
6. `git commit` (`.claude/rules.md`의 타입·`[AI-CONTEXT]` 형식 준수)
